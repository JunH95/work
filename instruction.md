# 깃허브 포트폴리오 최적화 및 학습 관리 지시서

## 1. 프로젝트 개요
- **목표**: '현대로템 AI 과정'의 학습 데이터를 체계적으로 정리하여 전문적인 기술 역량을 증명하는 깃허브 포트폴리오 구축.
- **핵심 가치**: "무엇을 배웠는가(이론)" + "어떻게 구현했는가(코드)" + "어떻게 해결했는가(트러블슈팅)".

## 2. 깃허브 관리 및 포트폴리오화 전략

### A. 학습 여정의 문서화 (Learning Log)
- 각 폴더 내에 `Learning_Log.md`를 생성하거나 기존 주석을 강화할 것.
- 단순히 코드 설명뿐만 아니라, 해당 주차에 배운 **핵심 이론(예: Flask의 동작 원리, REST API의 개념 등)**을 요약하여 포함할 것.
- 깃허브 잔디(Commit Graph)를 넘어, 커밋 메시지 자체가 학습 일지가 되도록 할 것.

### B. 주요 프로젝트 하이라이트 (Project Showcasing)
- `defense-stock-dashboard`와 같이 완성도가 높은 프로젝트는 별도의 섹션으로 분리할 것.
- 각 프로젝트 폴더 안에 **개별 README.md**를 작성하여, 프로젝트의 기획 의도와 본인이 기여한 핵심 로직을 상세히 적을 것.

### C. 의미 있는 커밋 메시지 규정 (Commit Convention)
- 클로드가 작업을 수행할 때마다 아래 규칙에 맞춰 커밋을 생성하게 할 것.
  - `feat`: 새로운 학습 예제 추가
  - `docs`: 이론 설명 주석 및 문서 수정
  - `refactor`: 중복 코드 제거 및 폴더 구조 변경
  - 예: `docs: Flask 세션 관리 이론 주석 추가 및 README 업데이트`

### D. 깃허브 프로필 및 메인 README 구성
- 저장소 메인 `README.md` 상단에 **'현대로템 AI 과정 핵심 기술 스택'** 배지를 추가할 것.
- 전체 폴더 구조를 트리 형태로 시각화하여 어떤 순서로 학습이 진행되었는지 보여줄 것.

## 3. 세부 수행 과제 (기존 과제 포함)
1. **폴더 재구조화**: `pro1` -> `Python_Foundation`, `pro4flask` -> `Backend_Flask` 등 직관적 명칭으로 변경.
2. **코드 다이어트**: 중복된 코드나 불필요한 테스트 파일은 과감히 삭제하거나 `archive/` 폴더로 이동.
3. **이론 주석 자동화**: 클로드의 분석 능력을 활용해 각 코드의 핵심 알고리즘과 이론 배경을 한글로 주석 처리.
4. **CLAUDE.md 구축**: 위 규칙들이 앞으로의 수업 파일에도 자동 적용되도록 가이드라인 작성.

## 4. 실행 지침
- 모든 작업은 `plan` 모드에서 시작하여 구조 변경안을 먼저 제시할 것.
- 깃허브에 올릴 때 개인정보(DB 접속 정보 등)가 포함되지 않도록 `.gitignore`를 철저히 검토할 것.

### E. 환경 설정 및 보안 관리
- 작업을 마칠 때마다 `.gitignore`를 점검하여 불필요한 파일이 포함되지 않도록 자동 업데이트할 것.
- 특히 `.claude/`, `.venv/`, `__pycache__/` 등이 누락되지 않았는지 상시 확인할 것.
